Automated bond covenant analysis skill

Problem

Extracting covenant terms from a bond Offering Memorandum is slow, detail-heavy work. Each OM runs to hundreds of pages, the key restricted-payment, debt-incurrence and structural-subordination terms are scattered throughout, and a single missed carve-out can change a credit view. Doing this by hand for every new deal is a major drain on analyst time, and consistency across analysts is hard to maintain.

What I built

A reusable Claude skill that reads a bond OM/OC and produces a structured Lucror Indenture Summary. It extracts covenant terms into a standardised set of fields, applies a per-field grading rubric to score covenant quality, and outputs the result as Word and Excel reports. It also generates an interactive HTML verifier that links every extracted field back to the exact OM page and verbatim quote, plus an automated quality-control pass that confirms all fields are populated and that each quote genuinely appears on the cited page. The skill is designed for Asia and Middle East credits, with structural checklists for holdco structures, keepwell enforceability and structural subordination.

Stack
Claude Python Anthropic API
Result

Turns a manual, multi-hour covenant read into a structured, graded and source-verified summary produced in one pass, with every term traceable back to the OM for review.

Real-time call transcription and summary tool

Problem

Transcription is not readily available for many calls credit analysts rely on. Earnings calls for unlisted companies, NetRoadshow presentations, audio calls with company management, and Zoom or Teams meetings with external parties often have no transcript. Capturing them means a human note-taker and hours of manual write-up.

What I built

A Python program that records live audio input and output and transcribes it in real time using AssemblyAI speech-to-text. The transcript is sent automatically to Claude, which adds speaker labels and produces a clean call summary. Output is a labelled transcript plus a ready-to-use meeting summary, with no manual note-taking.

Stack
Python AssemblyAI Claude API
Result

Removed the need for a human note-taker on management calls and non-recorded presentations. Frees the analyst to focus on the call itself, and delivers a summary the moment the call ends.

Financial Modelling using Claude

Problem

Building a full 3-statement model by hand — 10 years of historicals plus a 3-year forecast — is slow, repetitive work that eats into research time.

What I built

A complete 3-statement financial model for China Oil & Gas, following a standard template, with Claude handling data input and projections end-to-end under human guidance.

Stack
Claude Google Sheets
Result

10 years of historical data and a 3-year forecast, fully populated and reviewable in the linked model.
